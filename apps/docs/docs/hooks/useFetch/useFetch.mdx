## API 

Values used for initial state and internal settings are provided as a config
object 

### Config Example

```ts
const response = useFetch(<url>, <options>)
```


### Return Values

<HeadlessTable code items={[
    ["value",'any', 'The current state of the fetch request (response.data)'],
    ["loading",'boolean', 'True if the request is currently loading'],
    ["loaded",'boolean', 'True if the request was made successfully'],
    ["error",'null | any', 'Null if successful, error object if an error occurred'],
    ["reload",'Function','Clear state and send fetch request again'],
    ["reset",'Function','Reset internal state to initial values'],
]}/>


### Interfaces
```ts
export interface I_Fetch {
    watch?: any[];
    expire?: number;
    options?: object;
    initialData?: any;
    autoLoad?: boolean;
}
```

## Hook Methods

### reload
Reset the internal state and send the fetch request again.

```ts
() => void 
```
```ts
const res = useFetch('https://jsonplaceholder.typicode.com/todos')

res.reload()
```

<br />


### reset
Reset the internal state.

```ts
() => void 
```
```ts
const res = useFetch('https://jsonplaceholder.typicode.com/todos')

res.reset()
```

<br />

## Internal

This hook internally uses the `useState` hook to store the array and provides 
a wrapper around common array modification methods. 

Methods in `useStateArray` do not provide access to the current state for callback 
functions.
